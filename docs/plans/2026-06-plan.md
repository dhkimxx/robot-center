# 2026-06 Monthly Plan

## Scope

기간: 2026-06-08 ~ 2026-06-30

6월 목표는 P0 기능 전체를 제품 수준으로 완성하는 것이 아니라, **P0 기능을 관제 PoC로 시연 가능한 thin-slice 형태로 완성**하는 것이다.

이번 달의 성공 기준은 다음 흐름을 끝까지 보여주는 것이다.

```text
임무 생성
-> 로봇 1~2대 연결
-> RGB/Thermal/GPS/센서 표시
-> 이벤트/알람 기본 표시
-> 녹화 중 상태 표시
-> 임무 종료
-> 녹화 리플레이 재생
-> 로봇팀 연동 문서 제공
```

## June Goals

1. 로봇팀 연동에 필요한 Robot Gateway/WebRTC 계약을 PoC 기준으로 확정한다.
2. 다중 로봇 임무 관제와 선택 로봇 영상/센서 표시를 안정화한다.
3. 임무 종료 후 녹화 파일이 마감되고 리플레이되는 흐름을 보장한다.
4. 센서 데이터가 늘어나도 저장/표시가 깨지지 않는 기본 구조를 완성한다.
5. 이벤트/알람과 AI/SOP는 제품 수준이 아니라 PoC placeholder 또는 rule/mock 기반으로 구현한다.

## Week 1: Robot Contract And Live Pipeline

기간: 2026-06-08 ~ 2026-06-14

목표: 로봇팀이 붙을 수 있는 관제 연동 계약과 live pipeline을 고정한다.

작업:

- Robot Gateway API 정리
  - heartbeat
  - active mission 조회
  - SFU signaling URL
- WebRTC role별 endpoint 계약 정리
  - `/sfu/robot/ws`
  - `/sfu/operator/ws`
  - `/sfu/recorder/ws`
- Track/DataChannel 규칙 정리
  - `track.video_1`
  - `track.video_2`
  - `track.audio_1`
  - `channel.telemetry`
  - `channel.spatial`
  - `channel.event`
- Python/GStreamer Mock Robot 검증 흐름 정리
- `docs/stable/robot-interface.md`와 Swagger/OpenAPI 정합성 확인

완료 기준:

- Mock Robot이 stable 문서 기준으로 관제에 붙을 수 있다.
- 관제 화면에서 선택한 로봇의 RGB/Thermal/GPS/센서가 표시된다.
- `A-SYS-03`, `I-COM-02`, `I-COM-04` 진행 근거가 문서에 반영된다.

## Week 2: Recording Replay And Sensor PoC

기간: 2026-06-15 ~ 2026-06-21

목표: 녹화/리플레이와 센서 저장/표시를 PoC 시연 가능한 수준으로 안정화한다.

작업:

- 임무 종료 시 마지막 recording chunk finalize 보강
- recorder-worker job 동시성/중복 처리 검증
- MP4 muxing 실패/재시도 정책 정리
- MinIO object 상태와 DB recording 상태 정합성 확인
- replay UI에서 robot/session/chunk/file 구조 정리
- SensorDescriptor/SensorSample 기반 latest sensor 표시 보강
- GPS, battery, gas, temperature, IMU/fallback 센서 표시 정리

완료 기준:

- 임무 시작 -> 로봇 송출 -> 녹화 -> 임무 종료 -> 리플레이 재생이 통과한다.
- 마지막 chunk가 `녹화 중`으로 남지 않는다.
- 센서 종류가 늘어나도 UI가 깨지지 않는다.
- `D-SRV-07`, `E-DB-03`, `E-DB-06` 진행 근거가 문서에 반영된다.

## Week 3: P0 PoC Feature Thin-Slice

기간: 2026-06-22 ~ 2026-06-28

목표: P0 후보 기능을 PoC에서 보일 수 있는 최소 단위로 구현한다.

작업:

- 이벤트/알람 기본 표시
  - DataChannel event 또는 mock event 수신
  - 관제 화면 이벤트 로그 표시
  - 경고/주의 상태 기본 chip 표시
- 탐지 결과 시각화 thin-slice
  - 실제 AI 탐지 연동은 제외
  - mock detection event를 영상/이벤트 패널에 표시
- AI/SOP thin-slice
  - 실제 LLM 품질 검증은 제외
  - rule/mock 기반 상황 요약 또는 추천 패널 표시
- 관제 화면 UI 정리
  - 로봇 선택
  - 송출/녹화/연결 상태
  - 영상 overlay
  - 센서/이벤트 영역

완료 기준:

- 관제 화면에서 영상/센서/이벤트/기본 AI 요약 영역이 한 흐름으로 보인다.
- mock event/detection으로 P0 기능 방향성을 시연할 수 있다.
- `D-SRV-04`, `D-SRV-05`, `F-LLM-*` 중 실제 착수된 항목은 `planned -> in-progress` 후보로 검토한다.

## Final Days: Integration Demo And Documentation

기간: 2026-06-29 ~ 2026-06-30

목표: 6월 PoC 결과를 반복 시연 가능한 상태로 고정한다.

작업:

- 통합 검증
  - 로봇 1대
  - 로봇 2대
  - 관제자 1명
  - recorder-worker 포함
- 장애 상황 최소 검증
  - 로봇 재접속
  - 관제 화면 새로고침
  - recorder-worker 재시작
- 배포/시연 스크립트 정리
  - start
  - status
  - mock robot 실행
  - MinIO/PostgreSQL 확인
- 기능 문서 상태 재분류
  - planned
  - in-progress
  - done
- 6월 PoC 결과 리포트 작성

완료 기준:

- 한 번의 명령 흐름으로 서버/웹/recorder/mock robot 시연이 가능하다.
- 6월 PoC 검증 결과가 문서로 남는다.
- 7월 안정화/확장 작업 범위가 명확해진다.

## Out Of Scope For June

6월에는 다음 항목을 제품 수준으로 구현하지 않는다.

- 실제 긴급 제어 명령
- 실제 AI 탐지 모델 연동
- SOP 전체 자동 추천
- LLM hallucination 품질평가 체계
- 보안/권한/TLS/암호화 제품 수준 구현
- 3D/SLAM 고도화
- 실제 로봇 환경 완전 검증

## Expected Deliverables

- P0 PoC 시연 가능한 관제 화면
- 로봇팀 공유용 연동 가이드 갱신
- 안정화된 WebRTC live pipeline
- 임무 종료 후 리플레이 가능한 녹화 흐름
- 센서 확장 표시 구조
- 이벤트/알람/AI 요약 thin-slice
- 6월 PoC 검증 결과와 7월 작업 범위
